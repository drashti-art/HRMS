# **App Name**: WorkNest HR

## Core Features:

- Role-Based Authentication & Dashboards: Secure login system with JWT, redirecting users to dashboards customized for Super Admin, Admin, HR, Manager, or Employee roles based on assigned permissions.
- Employee Profile Management: CRUD operations for employee records by Admins and HR. Employees can view and update their personal profile information.
- Attendance & Leave Workflows: Employees can apply for leave and mark attendance. Managers and HR can approve or reject leave requests with an intuitive interface.
- Recruitment & Job Posting: HR personnel can create and manage job postings, track candidate applications, and facilitate onboarding processes.
- Basic Payroll Processing: Admins and HR can manage fundamental payroll data, ensuring accurate calculation and viewing of employee salaries.
- AI-Powered Resume Tool: A generative AI tool for HR to quickly summarize key information from candidate resumes, aiding in faster screening and matching to job requirements.
- Global Role & Permission Management: Super Admins can define, modify, and assign user roles and permissions across the entire system, ensuring flexible access control.

## Style Guidelines:

- Primary color: A deep, professional indigo (#2E2E6B). This hue anchors the visual design with a sense of authority and stability, ideal for an HR platform.
- Background color: A serene and understated pale lavender-grey (#F0F0F5). This provides a clean, breathable canvas that supports readability without being stark white.
- Accent color: A vibrant yet soft sky blue (#66B2E0). This bright accent provides contrast for calls-to-action and important information, creating an engaging user experience while remaining professional.
- Body and headline font: 'Inter' (sans-serif) for its modern, neutral, and highly readable characteristics, suitable for data-heavy HR applications.
- Employ a set of clean, minimalist outline icons. Icons should be easily understandable and maintain a consistent visual weight and style across all modules.
- Utilize a modular, card-based layout for dashboards and information display, emphasizing clarity, hierarchy, and ease of navigation across different roles and datasets. Implement responsive design principles.
- Incorporate subtle, functional animations for feedback on user interactions (e.g., button clicks, form submissions) and smooth transitions between pages and module sections.
